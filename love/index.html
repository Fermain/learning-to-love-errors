Since 'Error' is such a broad term, we can group them into types
Not all of these are relevant to a front end developer.
  • Relevant Errors
  • User Error:

    An application is working as intended, but the user is interacting with it in a way that was not expected.

    Example: I charged my laptop with live wires from the wall, and now it won't turn on!

  • Runtime Error:

    Something went wrong while the application was running.

    Example: Uncaught TypeError: .forEach does not belong on undefined

  • Build Error:

    Something went wrong the code for the application was being assembled.

    Example: code ELIFECYCLE errno 1 module not found
